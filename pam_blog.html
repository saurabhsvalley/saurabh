Purpose: This blog explains how to create a dynamic password history mechanism with PAM (Pluggable Authentication Modules). PAM currently allows configuring a static number of old passwords to prevent reuse, but does not natively support a dynamic model â€” especially where password changes are restricted to once per day.

  • Each time a password changes:
    • Backup the current opasswd to opasswd_old.
    • Update both opasswd and opasswd_old with the new password entry.
  • If the configured number of previous passwords changes (increased or decreased):
    • Reconstruct opasswd from opasswd_old to match the new required count.

    • Notes:

    • This algorithm works across all PAM versions that use opasswd and oshadow.
    • You can also leverage the chage utility for managing password expiry and aging.
    • Best practice: Always create a copy of opasswd, opasswd_old, and oshadow before making changes. Apply modifications on the copy, then rename it back to the original file for safety.
